Self-signed TLS/SSL certificate #67

Description

The server’s TLS/SSL certificate is self-signed. Self-signed certificates are not trusted by browsers and other HTTP clients, especially because TLS/SSL man-in-the-middle attacks typically use self-signed certificates to eavesdrop on TLS/SSL connections.

Fix guidance (60 minutes)

Create a new TLS/SSL certificate, request a trusted certificate authority (CA) to sign it and replace the self-signed certificate with the newly generated one.
